# Security Policy

## Scope

This repository contains a static marketing website. It must not contain customer data, secrets, credentials, private support evidence, proprietary test fixtures, or production configuration.

## Reporting

Until a dedicated security contact is published, do not open a public issue containing sensitive details. Contact the repository owner privately through the GitHub account associated with this repository.

## Website rules

- Never commit API keys, tokens, passwords, or private certificates.
- Use synthetic or fully sanitized screenshots.
- Do not add client-side collection scripts without documenting the data flow and updating the privacy notice.
- Do not embed downloadable installers until release signing, checksums, licensing, and support expectations are defined.
- Treat third-party scripts as supply-chain dependencies and minimize them.
